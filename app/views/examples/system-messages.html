This page demonstrates how to display system messages in the MOT system.

These must not be used as validation messages!

Use system messages to draw attention to important content on the page.

  1. Standard message
  2. Success message
  3. Failure message
  4. Important message

1. Standard message

Standard messages .message have no icon and are used to convey information that:

  • may, in some edge cases, help a user complete a task
  • provides an alternative path through a task (upcoming vehicle search edge cases)
{{> mot_m_system-message CONTEXT_STANDARD}}

An optional heading .message__heading can be used.

{{> mot_m_system-message CONTEXT_STANDARD_HEADER}}

2. Success message

“A good thing has happened.”

Success messages .message--success are used to convey information that:

  • shows a user has been successful in their interaction with the system

Success messages must not be used:

  • to communicate validation
  • on task completion pages (unless a message is required, or the information is supplementary to the task) - The content of the page should communicate the outcome
{{> mot_m_system-message CONTEXT_SUCCESS }}

An optional heading .message__heading can be used.

{{> mot_m_system-message CONTEXT_SUCCESS_HEADER}}

3. Failure message

“A bad thing has happened.”

Failure messages .message--failure are used to convey information that:

  • shows a user's interaction with the system has failed and they must essentially "stop and try again"

Failure messages must not be used:

  • to communicate validation
  • on task completion pages (unless a message is required, or the information is supplementary to the task) - The content of the page should communicate the outcome
{{> mot_m_system-message CONTEXT_FAILURE}}

An optional heading .message__heading can be used.

{{> mot_m_system-message CONTEXT_FAILURE_HEADER}}

4. Important message

“A bad thing could happen if you continue.”

Important messages .message--important are used to convey serious information that will often have consequences.

Use these messages with care.

{{> mot_m_system-message CONTEXT_IMPORTANT}}

An optional heading .message__heading can be used.
